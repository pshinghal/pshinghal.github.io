public class PSGTD implements GTD

My (simple) implementation of Getting Things Done

Inbox

Implemented as a Trello board, Basket.

Lists on the board correspond to various categories of work - Reading, Personal, School and NUS Hackers.

Cards correspond to action items. Some tasks contain checklists, some have due dates.

New items get added as cards into the appropriate list. Every day, daily objectives get moved into the Today board.

Today

Contains 3 lists - To Do, Doing & Done.

The cards to be processed each day get moved into the To Do list. They then move to Doing and Done. At the end of the day, all cards in Done get archived and stuff in To Do and Doing is moved back into the Basket. The objective is to have all of the day's cards end up in Done by the end of the day. Adjust daily objectives by measured output (and increase gradually).

Long Term

A number of tasks are very low-priority and possibly optional. These get added into the Long Term board.

The lists in this board are: Thoughts/Writing, Reading, Watching/Listening and Hacking. New lists may be added as required.

Sometimes, tasks from Basket get moved to this board.

This board is reviewed every once in a while (at least weekly), and tasks that no longer seem relevant, useful or interesting are archived. Tasks from this board are not currently done on a regular basis, but this needs to be changed. Ideally, at least one task from this should be moved into the Basket and done every week.
